本設計はEIGOモン・SAKUモンの既存アプリに、**新アプリを作らず**B2B管理レイヤーを追加するものです。

### プラン構成

| プラン | 月額 | 対象 | 内容 |
|---|---|---|---|
| 講師プラン | ¥1,980/月 | 個別指導講師・家庭教師 | 生徒のZPDスコア・フェーズ推移・弱点分析を閲覧・管理 |
| 教室プラン | ¥9,800/月（基本）+ ¥1,500/追加講師 | 学習塾・教育機関 | 塾全体の講師・生徒を統合管理。複数教師席 |

## 2. ZPD理論実装

### スコア計算式

```typescript
ZPDScore = accuracy × 0.5 + CV_stability × 0.2 + errorPattern × 0.3
```

### 重み付けの理論根拠（Delphi法的専門家合議）

| 指標 | 重み | 根拠文献 |
|---|---|---|
| accuracy（正答率） | 0.5 | Anderson (1983) ACT理論 / Corbett & Anderson (1994) 知識追跡モデル |
| CV_stability（反応時間変動係数） | 0.2 | Newell & Rosenbloom (1981) Power law of practice |
| errorPattern（系統的誤り） | 0.3 | Corder (1967) 誤り分析理論 / Tatsuoka (1983) Rule Space Model |

### フェーズ遷移システム

```typescript
// PHASE_TRANSITION_TYPE_ENUM（6値）
type PhaseTransitionType =
  | 'phase1_to_2'
  | 'phase2_to_3'
  | 'phase2_to_1'      // ロールバック
  | 'phase3_to_2'      // ロールバック
  | 'phase1_to_2_recovery'
  | 'phase2_to_3_recovery'

// フェーズ定義
Phase1: 外発的動機（External）  → 報酬・ポイント中心
Phase2: 橋渡し（Bridge）        → 外発+内発の混在期
Phase3: 内発的動機（Internal）  → 自己目標・成長実感
```

### ヒステリシス付きフェーズ遷移

連続3日以上の条件充足で遷移。単発の変動では遷移しない。

### ロールバック頻度ガード

```typescript
PHASE_TRANSITION_FREQUENCY_GUARD: {
  trigger: 'rollbackCount >= 3 in 30 days',
  suspensionScope: 'per_student',
  timeoutPolicy: {
    maxSuspensionDays: 14,
    automaticResume: false,  // 教師の明示的操作が必要
    overdueAlert: '14日後にin-app通知',
  },
  resumeChecklist: [
    { id: 1, label: '直近30日のロールバック原因を確認した', required: true },
    { id: 2, label: '保護者または生徒本人に確認した', required: false,
      skipReasonField: { required_if_skipped: true, loggedIn: 'zpd_suspension_logs' } },
    { id: 3, label: '今後の学習計画を調整した', required: true },
  ],
}
```

### キャリブレーション

- minimumSampleSize: 30 per_phase_transition_type
- M6実証的検証: ロジスティック回帰による重み推定
- AUCしきい値: Hosmer & Lemeshow (2000) 準拠
- Wilson CI: Agresti-Coull (1998)、≥20データポイント要件
